About This Toolkit

This toolkit combines practical security tools with real network analysis capabilities. The security tools use real cryptographic functions (bcrypt, SHA-256) to help manage passwords and validate data. The breach checker uses the k-anonymity model and the XposedOrNot API to check for leaked credentials without ever exposing them. The network tools use Python's socket library and Scapy for real TCP port scanning and live packet capture — both require the local agent running with elevated privileges on your machine.

Password Strength Assessor

Evaluate how strong any password is — instantly

Type or paste any password and get an immediate strength rating with specific feedback on what to improve. The assessor checks against common password lists and dictionary words before scoring.

Password Strength Analyzer

How to use it

  1. 1
    Navigate to Analyze under Milestone 1 in the sidebar.
  2. 2
    Type your password into the input field.
  3. 3
    Click Assess Password. Your result appears instantly below.
  4. 4
    Read the feedback and improve your password as suggested.

Scoring breakdown

RatingScoreWhat it means

One point is awarded for each of: 12+ characters, uppercase letters, lowercase letters, digits, and special characters. Passwords found in a common-password list or containing a dictionary word are immediately rated Weak.

Password Generator

Create cryptographically secure passwords with one click

Generate a random, secure password between 4 and 128 characters. Every result is guaranteed to contain uppercase, lowercase, digits, and special characters. SHA-256 and bcrypt hashes are included for developers who need to store credentials securely.

Password Generator

How to use it

  1. 1

  2. 2
    Set your desired password length.Default is 16. Valid range is 4–128 characters.
  3. 3
    Click Generate Password.
  4. 4
    Copy the password from the result card.SHA-256 and bcrypt hashes are also shown — copy them if you need them for storage.
Save Generated Password
Privacy note

The server logs only the hashes — your plaintext password is never stored anywhere.

SHA-256 Hash

  • Fixed 64-character hex string
  • Fast, commonly used for checksums
  • Not recommended alone for storing user passwords

bcrypt Hash

  • 12-round work factor
  • Salted — every hash is unique
  • Industry standard for password storage

Form Input Validator

Sanitize and validate web form inputs against XSS and SQL injection

Paste any text — a name, email, username, or message — and the validator runs it through a two-phase pipeline: sanitization first (stripping dangerous code), then format validation (checking field-specific rules).

Form Input Validator

How to use it

  1. 1
    Navigate to Validate under Milestone 1 in the sidebar.
  2. 2
    Select the field type from the dropdown: name, email, username, or message.
  3. 3
    Enter your text in the input box.
  4. 4
    Click Validate. The result shows whether the input is valid, the sanitized output, and any error details.

Validation rules by field type

FieldRules
Full NameMin 2 chars · Letters, spaces, hyphens, apostrophes only · No digits
EmailNo spaces · Must contain @ · Must end with a valid domain suffix
Username4–16 chars · Cannot start with a digit · Letters, digits, underscores only
MessageNon-empty · Max 250 chars · No script tags, suspicious attributes, or SQL keywords
SQL injection detection

If a SQL keyword or injection pattern is detected in any field type, the input is immediately blocked and flagged — it never reaches the format validator.

Leaked Password & Breach Checker

Check if your password or email appeared in known data breaches

The Breach Checker lets you find out whether a password or email address has been exposed in a publicly known data breach — without ever sending your actual credentials over the network. It has two independent tabs: Password Check and Email / Username.

k-Anonymity — your password never leaves your device

Only the first 5 characters of your password's SHA-1 hash are transmitted to the API. The rest of the matching is done locally in your browser, so your plaintext password is never sent over the network.

Password Check tab

Breach Checker — Password Check

This tab queries the Have I Been Pwned Passwords API using the k-anonymity model. It tells you how many times your password has appeared in known breach databases — even once is a strong reason to change it immediately.

  1. 1
    Go to Breach under Milestone 1 in the sidebar.
  2. 2
    Make sure the Password Check tab is selected (it is the default).
  3. 3
    Type your password into the Enter Password field. Use the eye icon on the right to show or hide the characters.
  4. 4
    Click Check Password. The result shows exactly how many times (if any) that password has appeared in breach datasets.

Email / Username tab

Breach Checker — Email / Username

This tab is powered by the XposedOrNot free, open-source breach API — no API key is required. Enter any email address or username to see a list of known breaches it has appeared in.

  1. 1
    Click the Email / Username tab at the top of the tool.
  2. 2
    Enter an email address or username in the input field.
  3. 3
    Click Check Email. Results list every known breach the address has appeared in, along with breach details.

Password Check

  • Uses Have I Been Pwned Passwords API
  • k-Anonymity: only first 5 SHA-1 hash chars sent
  • Matching done locally in your browser
  • Returns exact breach exposure count

Email / Username Check

  • Powered by XposedOrNot API
  • Free and open-source — no API key needed
  • Returns list of named breach sources
  • Works with both emails and usernames
What to do if a breach is found

Change the affected password immediately and never reuse it on any other site. Enable two-factor authentication where available, and use the Password Generator to create a new, unique password for each account.

Port Scanner

Scan TCP ports on any host and classify open services by risk

The Port Scanner performs a TCP connect scan — it attempts to open a connection to each port and reports whether it's open, closed, or filtered. Up to 100 ports run in parallel for fast results. Open ports are automatically classified by risk level.

Enabling the Local Agent

Both MS2 tools (Port Scanner and Traffic Analyzer) require the local agent running on your machine. When you first open the tool, you will see the setup panel below. Click Enable Local Scanning to connect.

Network Tools — Initial Setup Panel

If the agent is not yet running on your machine, the panel will show a Local Agent Not Detected message. Start the agent first (see the testing guide), then click the refresh icon to recheck.

  2. 2

  3. 3

Port Status

  • OPEN — connection successful, service listening
  • CLOSED — connection refused or timed out
  • FILTERED — network error, likely a firewall

Risk Levels (open ports)

  • High — Telnet 23, FTP 21, RDP 3389, SMB 445
  • Medium — SSH 22, DNS 53, SMTP 25
  • Low — HTTP 80, HTTPS 443
Authorized use only

Port scanning without permission is illegal in many jurisdictions. Only scan systems you own or have written authorization to test.

Live capture requires the local agent

Live packet capture requires root access that is unavailable on the cloud deployment. Run the local agent with sudo (Linux/macOS) or as Administrator (Windows) to enable real capture. Without elevated privileges the stream and snapshot buttons remain disabled.

Troubleshooting

Solutions to the most common issues

The first request is very slow or times out

The app is hosted on Render's free tier, which spins down after inactivity. The first request after a cold start can take 30–60 seconds. Wait for the response before retrying — clicking again only queues a second request behind the first.

"Local Agent Not Detected" after clicking Enable Local Scanning

  1. 1
    Use Chrome or Edge. Firefox blocks connections from HTTPS pages to http://127.0.0.1 — this is a browser security policy, not a bug. Firefox users will always see "Not Detected" and cannot use the local agent.
  2. 2
    Brave users: Brave Shields blocks the agent probe even in normal mode. Click the Brave lion icon in the address bar, toggle Shields off for securekit-whk3.onrender.com, then reload.
  3. 3
    Confirm the agent is running. The terminal window running the agent must stay open. If it closed, re-run StartAgent.sh / StartAgent.bat.
  4. 4
    Verify the agent is reachable:curl http://127.0.0.1:8765/health — a JSON response means it is running. "Connection refused" means the agent stopped or failed to bind.
  5. 5
    Check for a port conflict (another process already using port 8765):Linux/macOS: ss -tlnp | grep 8765  ·  Windows: netstat -ano | findstr :8765
    If something else is on 8765, stop it and restart the agent.

Agent banner shows "Scapy: not available"

The most common cause is running the agent from a partial copy of the repo — one that has local_agent.py but is missing the src/ directory. The agent requires src/web_security_tool/ to be present alongside it.

Download the full repository

Use the "Download Necessary Files" button in the setup panel to get the complete repo ZIP. Downloading only local_agent.py or the script files is not enough — the src/ package tree must be present next to them.

Stream and Snapshot buttons are grayed out even though the agent is connected

The agent connected successfully but is running without elevated privileges — or Scapy failed to import. The panel will show "Scapy unavailable — restart agent with sudo for live capture."

  • Linux / macOS: stop the agent and re-run StartAgent.sh. The script prompts for your sudo password automatically.
  • Windows: right-click StartAgent.bat and choose Run as Administrator.
  • Do not run sudo python3 local_agent.py directly — the system Python does not have the venv packages and Scapy will fail to import even with root.

CORS error visible in browser DevTools

The agent's allow_origins list may not include https://securekit-whk3.onrender.com. Open local_agent.py in a text editor, find the allow_origins list, verify the URL is present, and restart the agent. This usually happens when using an older copy of the file downloaded before this was fixed.

Download buttons in the setup panel show an error

The Render backend may be cold-starting. Wait 30–60 seconds and try again. Alternatively, download the files directly from the GitHub repository using the link in the setup panel.

Breach Checker returns no results or an error

The Password Check tab uses the Have I Been Pwned API and the Email tab uses XposedOrNot — both are external services. If either returns an error, check your internet connection first. Occasional API downtime is possible; wait a minute and try again. The breach checker does not require the local agent and works entirely in your browser for the password tab.
